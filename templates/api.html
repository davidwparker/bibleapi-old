The Bible API is a RESTful API.

What's this mean? That means that all requests are made using HTTP "GET" requests against a given URL for a Resource.

JSON

We only support the returning of JSON and have no plans to support returning XML.

Testing out the API using curl

Using Curl, you can test the API as follows:
curl --header "Accept: application/json" http://localhost:8000/bible/VERSION/BOOK/CHAPTER/VERSE-VERSE2/

Header and Variables

  • The Accept header is required and will return the website HTML if it is not present.
  • (required) VERSION, limits the results to only the version.
  • (required) BOOK, limits the results to only the book.
  • (required) CHAPTER, limits the results to only the chapter.
  • (optional) VERSE, if provided with CHAPTER, will limit the results to only the given verse.
  • (optional) VERSE2, if provided with CHAPTER and VERSE, will limit the results to a range of verses.
